1. Strategy Overview

1.1 What is Fibo 71 CP 2.0?

Fibo 71 CP 2.0 is an automated trading strategy that combines Fibonacci retracement analysis with Break of Structure (BOS) detection to identify high-probability reversal entries in forex markets. The "71" refers to the primary entry zone at the 71-79% Fibonacci retracement level, where deep pullbacks offer favorable risk:reward ratios.

1.2 Core Concept

ParameterValueDescription
TimeframeH4 (primary), H1/M15 (optional)Primary trading interval
Risk per Trade1% of account0.5% on correlated pairs
Entry Zone71%-79% Fib retracementDeep pullback zone (premium)
Take ProfitFib 0% (swing extreme)Previous swing high/low
Stop LossFib 100% (opposite extreme)Full invalidation level
Win Rate~70% (CP 2.0 with confirmation)~40% without confirmation (CP 1.0)

1.3 Strategy Variants

The system implements two closely related strategies:

CP 2.0 (Standard)

  • Direct BOS (Break of Structure) detection
  • Fibonacci calculated from swing points
  • Entry at 71-79% retracement (limit orders)
  • Single TP at Fib 0%, single SL at Fib 100%
  • Optional: Imbalance and Liquidity Sweep filters

BMS (Break of Market Structure)

  • More conservative approach with momentum confirmation
  • Extremum tracking after BMS detection
  • Entry at 62-71% retracement zone
  • 3 TP levels with partial closes (33%/33%/34%)
  • Liquidity sweep + confirmation candle required
  • 4 configurable filters (Trend, Volume, Volatility, R:R)

2. Core Algorithm

2.1 Complete Algorithm Flow

Step 1: DETECT BOS/BMS (Break of Structure) | |--> Bullish: close > swing_high (with momentum) |--> Bearish: close < swing_low (with momentum) | Step 2: FIND SWING POINTS | |--> For BUY: Swing Low (start) -> Swing High (end) |--> For SELL: Swing High (start) -> Swing Low (end) | Step 3: CALCULATE FIBONACCI LEVELS | |--> 0% = TP (Take Profit) |--> 38.2%, 50%, 61.8% = Mid levels |--> 71% = Entry zone start |--> 79% = Entry zone end |--> 100% = SL (Stop Loss) | Step 4: WAIT FOR RETRACEMENT | |--> Price pulls back into 71-79% zone |--> Place LIMIT ORDER at zone | Step 5: OPTIONAL FILTERS | |--> Imbalance (IPA) check |--> Liquidity Sweep check |--> Confirmation candle | Step 6: EXECUTE TRADE | |--> Entry: Limit order in 71-79% zone |--> TP: Fib 0% (previous swing) |--> SL: Fib 100% (opposite swing) |--> Risk: 1% of account balance

2.2 Break of Structure (BOS) Detection

The BOS detector identifies when price breaks through a previous swing point, signaling a potential trend continuation or reversal setup.

Swing Point Detection

Swing High: high[i] is greater than the surrounding N candles (default N=5)
Swing Low: low[i] is less than the surrounding N candles (default N=5)
ParameterDefaultDescription
lookback50Number of bars to scan for swing points
swing_period5Bars on each side for swing confirmation

BOS Conditions

Bullish BOS: Current candle closes above the most recent swing high.

Bearish BOS: Current candle closes below the most recent swing low.

2.3 Break of Market Structure (BMS) Detection

BMS is a stricter variant requiring momentum confirmation:

ParameterDefaultDescription
swing_lookback5Window for swing point detection
momentum_candles3Minimum consecutive candles in direction
body_threshold0.60Body must be >= 60% of candle range
distance_atr0.5Minimum break distance in ATR multiples
Momentum Validation: At least 3 consecutive candles must move in the same direction, each with body >= 60% of the total range. The break distance must exceed 0.5 * ATR(14).

2.4 Extremum Tracking (BMS Strategy)

After BMS is detected, the system tracks the highest high (bullish) or lowest low (bearish) until price retraces back through the BMS level. This confirmed extremum becomes the Fibonacci anchor point.

BULLISH EXAMPLE: 1. BMS detected (close > swing_high at 43,250) 2. Price continues up to 43,850 (track highest high) 3. Price drops back below 43,250 (BMS level) 4. Confirmed extremum = 43,850 5. Fibonacci: swing_low (42,800) --> 43,850
ParameterDefaultDescription
min_candles1Minimum candles before confirmation
timeout_candles50Maximum candles to wait for confirmation

2.7 Liquidity Sweep Detection

A liquidity sweep occurs when price briefly pushes beyond a key level (triggering stop losses) before reversing. This is detected via wick-to-body ratio analysis.

BULLISH SWEEP EXAMPLE: +---+ | | <-- Body = 10 | | +---+---+---+ | | <-- Lower Wick = 25 +-----------+ Wick Ratio = 25 / 10 = 2.5x (VALID)
ParameterDefaultDescription
min_wick_to_body_ratio2.0Minimum wick/body ratio for sweep detection
ideal_wick_to_body_ratio3.0Ideal ratio for high-confidence sweeps

Bullish Sweep: Lower wick >= 2x body size, with price closing above the Fib 62% level.

Bearish Sweep: Upper wick >= 2x body size, with price closing below the Fib 62% level.

2.8 Confirmation Candle

After a liquidity sweep, the system waits for a confirmation candle:

  • Direction: Must align with the BOS/BMS direction (green for bullish, red for bearish)
  • Body size: Body >= 50% of the total candle range
ParameterDefaultDescription
body_percent0.50Minimum body as % of range

3. Entry Filters

3.1 Imbalance (IPA) Filter

Detects price gaps (Imbalance/Imbalance Price Action) between candles, indicating strong directional momentum.

BEARISH IMBALANCE: Candle 1 Candle 2 Candle 3 | | | v v v +---+ +---+ +---+ | | | | | | | | | | | | +---+ +---+ +---+ |-------GAP-------| Low > High <-- GAP exists = BEARISH
ParameterDefaultDescription
enable_imbalancefalseEnable/disable IPA filter
min_imbalance_pips5.0Minimum gap size in pips

3.2 Trend Filter (EMA 50/200)

Confirms that the trade direction aligns with the overall trend.

DirectionCondition
BullishEMA(50) > EMA(200) - Uptrend
BearishEMA(50) < EMA(200) - Downtrend
When to enable: Trend following strategies, longer timeframes (H1, H4), beginner traders.
When to disable: Scalping, mean reversion, consolidation markets.

3.3 Volume Filter

Ensures sufficient trading volume to validate the move.

Condition: Current Volume > Average Volume (20 periods)

3.4 Volatility Filter (ATR)

Avoids trading in low-volatility periods.

Condition: ATR(14) > Average ATR(20)

3.5 Risk:Reward Filter

Ensures every trade meets a minimum reward-to-risk ratio.

R:R = (TP - Entry) / (Entry - SL)
Example: Entry=100, SL=98, TP=104 -> R:R = 4/2 = 2:1
PresetMin R:R
Conservative2.5:1
Balanced2.0:1
Aggressive1.5:1

4. Risk Management

4.1 Position Sizing

Position size is calculated automatically based on account balance, risk percentage, and stop loss distance:

Lot Size = (Account Balance * Risk%) / (SL Distance in pips * Pip Value)

Example:
  Balance: $10,000
  Risk: 1%
  SL Distance: 50 pips
  Pip Value: $10 (standard lot)

  Lot = ($10,000 * 0.01) / (50 * $10) = 0.02 lots

4.2 Stop Loss Placement

SL is placed at Fibonacci 100% level with an optional buffer to avoid stop hunting:

BULLISH: SL = Fib 100% * (1 - buffer%)
BEARISH: SL = Fib 100% * (1 + buffer%)

Default buffer: 0.1%

6. Grid Order System

The BMS strategy supports grid orders - multiple limit orders distributed across the Fibonacci entry zone for better average entry price.

6.1 Grid Configuration

ParameterDefaultDescription
enable_gridtrueEnable grid orders
grid_count5Number of orders in the grid
grid_spacingequalSpacing mode: equal, fib, custom

6.2 Spacing Modes

ModeDescriptionExample (5 orders in 62-71% zone)
equalEqual distance between orders62%, 64.25%, 66.5%, 68.75%, 71%
fibFibonacci-based spacingDenser near deeper levels
customUser-defined levelsCustom array of Fib levels

6.3 Risk Distribution

Risk is distributed evenly across all grid orders. Each order carries (total_risk / grid_count) risk. When multiple orders fill, the system tracks the average entry price and adjusts TP/SL accordingly.

7.2 Python Implementation Details

CP2Strategy (src/strategies/fibo71_strategy.py)

Main orchestrator that:

  1. Calls BOSDetector.detect_bos() to find setups
  2. Calculates Fibonacci levels via FibonacciCalculator
  3. Checks risk with RiskManager.can_open_trade()
  4. Creates TradeSetup dataclass with entry/SL/TP
  5. Places limit orders via MT5 API

BMSFiboLiquidityStrategy (src/strategies/bms_fibo_liquidity_strategy.py)

State machine-based strategy with the following states:

IDLE --> BMS_DETECTED --> TRACKING_EXTREMUM --> EXTREMUM_CONFIRMED
  |                                                     |
  |--> IN_FIB_ZONE --> SWEEP_DETECTED --> IN_TRADE       |
  |                                                     |
  +-----------------------------------------------------+

BOSDetector (src/indicators/bos.py)

  • _find_swings() - Finds swing highs/lows (configurable period)
  • _check_imbalance() - Detects IPA (gap between candle 1 and 3)
  • _check_liquidity() - Detects false breakout before BOS
  • Returns BOSResult dataclass with direction, swing points, confidence

RiskManager (src/risk/manager.py)

  • 1% risk per trade, 0.5% on correlated pairs
  • CORRELATION_GROUPS dict maps currencies to correlated pairs
  • calculate_lot_size() based on SL distance and risk %
  • Max daily trades and open positions limits
